## ADDED Requirements
### Requirement: 订单创建
系统 SHALL 支持创建新的采购订单，允许客户选择多个产品和数量。

#### Scenario: 成功创建订单
- **WHEN** 客户提交包含产品列表和配送地址的订单请求
- **THEN** 系统创建订单并返回订单 ID
- **AND** 确认订单已提交到阿里巴巴平台

#### Scenario: 订单创建失败
- **WHEN** 客户提交的订单信息不完整或库存不足
- **THEN** 系统返回详细的错误信息
- **AND** 不创建任何订单记录

### Requirement: 订单列表查询
系统 SHALL 提供客户订单列表查询功能，支持按状态筛选。

#### Scenario: 获取所有订单
- **WHEN** 客户请求订单列表
- **THEN** 系统返回该客户的所有订单摘要信息
- **AND** 包含订单 ID、创建时间、总金额、状态等信息

#### Scenario: 按状态筛选订单
- **WHEN** 客户指定特定的订单状态进行查询
- **THEN** 系统返回符合状态条件的订单列表

### Requirement: 订单详情查询
系统 SHALL 提供单个订单的详细信息查询功能。

#### Scenario: 获取订单详情
- **WHEN** 客户通过订单 ID 查询订单详情
- **THEN** 系统返回完整的订单信息
- **AND** 包含订单项、配送地址、费用明细、物流追踪信息

#### Scenario: 订单不存在
- **WHEN** 客户查询不存在的订单
- **THEN** 系统返回订单未找到的错误信息

### Requirement: 物流追踪链接
系统 SHALL 提供订单物流追踪的外部链接功能。

#### Scenario: 获取物流追踪链接
- **WHEN** 客户请求订单的物流追踪信息
- **THEN** 系统返回外部物流追踪链接
- **AND** 包含物流公司名称和追踪 URL

#### Scenario: 无物流信息
- **WHEN** 订单暂无物流信息
- **THEN** 系统返回暂无追踪信息的提示
